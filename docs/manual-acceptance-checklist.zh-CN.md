# WR.Next 人工验收清单

## 目标

把当前宿主收紧到一个能人工直接验收的形态：

1. 能启动
2. 能看见进程接管状态
3. 能显式接管 / 断开
4. 能关闭干净

## 验收前提

- 运行目录已由 `build-uihost.bat` 或等价构建链生成
- 当前只验收单进程，不测多进程
- 只测 `WR.OriginalUiHost`

## 验收步骤

### 1. 启动

- 运行 `run-uihost-latest.bat`
- 观察宿主是否正常打开
- 观察是否先出现健康检查错误
- 若已有旧宿主实例未退出，本次启动必须被健康检查阻断并给出明确提示

通过：

- 宿主窗口可见
- 没有阻断性报错框

### 2. 进程管理页

- 进入“进程管理”页
- 观察候选进程列表是否有真实 WoW 进程
- 观察每行是否能看到：
  - PID
  - 窗口标题
  - 角色
  - 等级
  - 血量
  - 坐标
  - 接管门
  - 状态

通过：

- 列表非空时，接管门和状态文本正常刷新
- 进程条目按钮显示“接管”或“断开”

### 3. 显式接管

- 点击某一条目的“接管”
- 观察按钮是否切换为“断开”
- 观察接管门是否从“未接管”进入“接管中 / 未附着 / 内存失败 / Hook未就绪 / 未进世界 / 已丢失 / 故障 / 已通过”之一
- 观察状态是否从“未接管”进入接管中 / 已接管 / Ready 一类状态

通过：

- 按钮态、接管门和状态文本一致
- 不会自动改接到别的 PID
- `product-chain-actions.txt` 中的 `session-state-transition` 与行内“接管门/状态”语义一致

### 4. 切页复用

- 接管后切换到别的页面再切回来
- 再回到进程管理页刷新一次

通过：

- 当前 PID 仍是同一个
- 不会因为切页触发重新接管

### 5. 关闭

- 关闭宿主窗口
- 等待进程退出

通过：

- 宿主进程退出
- 下一次启动不继承旧会话脏状态

## 当前已知限制

- 这是单进程验收，不覆盖多进程管理
- 当前不做全量战斗行为验收
- 当前不把插件页、工具页作为验收重点
