# WR.Next 首推前预热整理

## 结论

当前仓库在首次推送前，最需要处理的不是功能补丁，而是仓库边界。

已经确认的首要问题有三类：

1. 目录尚未初始化为 Git 仓库
2. 本地运行产物与源码资料混放
3. 缺少根级仓库说明与忽略规则

因此这次预热整理的目标是：

- 让仓库首推更干净
- 让后续提交范围更稳定
- 避免把大量个人环境和运行垃圾带进远端

## 已确认事实

### 1. 当前目录还不是 Git 仓库

已验证：

```text
git -C D:\666\work\WR.Next status --short
fatal: not a git repository (or any of the parent directories): .git
```

说明当前只是“准备推送的目录”，还没完成本地仓库初始化。

### 2. 当前目录存在大量不适合首推跟踪的内容

已确认包括：

- `artifacts/uihost-runtime-layout`
- `artifacts/截图`
- `tools/**/bin`
- `tools/**/obj`
- `src/**/bin`
- `src/**/obj`
- 本地 `Settings/Logs`

这些内容多数是：

- 构建输出
- 运行时聚合产物
- 诊断工具产物
- 本地截图
- 个体机器配置漂移

### 3. 构建入口已经具备统一脚本

当前可复用入口已经存在：

- `build-uihost.bat`
- `run-uihost-latest.bat`
- `tools/layout/Publish-OriginalUiHostLayout.ps1`

所以首推前不需要再额外发明新的入口脚本，优先把仓库说明补完整即可。

## 本次整理落点

### 1. 新增根级 `README.md`

作用：

- 说明仓库目标
- 说明主项目位置
- 说明目录分工
- 说明当前边界
- 说明构建运行入口

### 2. 新增根级 `.gitignore`

作用：

- 排除 `bin/obj`
- 排除运行产物
- 排除截图和日志
- 排除诊断工具编译垃圾
- 保持首推范围收敛在源码、文档、脚本、对照资料

### 3. 保留 `docs` 作为阶段结论文档区

这符合当前项目的推进方式：

- 每次一包
- 每包一文
- 结论先行

## 当前建议的首推范围

建议首推只包含：

1. `src`
2. `tools`
3. `docs`
4. `external`
5. `README.md`
6. `.gitignore`
7. 根级批处理脚本

不建议首推包含：

1. `artifacts/uihost-runtime-layout`
2. 本地截图
3. 运行日志
4. 任意 `bin/obj`
5. 个人设置样本

## 为什么这样整理

原因很简单：

1. 首推越干净，后续越容易审查改动
2. 运行产物一旦进仓库，后续每次提交都会被噪音淹没
3. 当前仓库的真正价值在源码、接管逻辑、逆向对照资料、中文结论文档，而不在本地产物快照
